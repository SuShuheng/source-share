### 17. LLM 多实例处理性能退化：实例数量 vs. 上下文长度

**来源**: [arXiv:2603.22608](https://arxiv.org/abs/2603.22608) | **时间**: 2026-03-25

- **研究问题**: LLM 处理多文档/多实例时的性能表现
- **发现**:
  - 小规模实例（20-100）：轻微性能退化
  - 大规模实例：性能崩溃
  - **关键结论**: 实例数量比上下文长度对性能影响更大
- **优化建议**: 同时关注上下文长度和实例数量

**实践价值**: 为优化 LLM 多实例处理性能提供指导。

### 18. Bridging the Know-Act Gap: 通过任务级自回归推理

**来源**: [arXiv:2603.22619](https://arxiv.org/abs/2603.22619) | **时间**: 2026-03-25

- **核心问题**: LLM 对错误输入生成看似有效的答案（"知行差距"）
- **原因**: token 级自回归将任务选择与内容生成纠缠
- **解决方案**: DeIllusionLLM
  - 任务级自回归框架
  - 显式建模决策（验证 vs. 回答）
  - 自蒸馏统一判别判断和生成推理
- **效果**: 大幅减少"尽管有错误仍回答"的失败，保持通用推理性能

**价值**: 自蒸馏是弥合判别-生成知行差距的有效可扩展方案。

### 19. MuQ-Eval: AI 音乐生成的开源单样本质量指标

**来源**: [arXiv:2603.22677](https://arxiv.org/abs/2603.22677) | **时间**: 2026-03-25

- **问题**: Fréchet Audio Distance 无法评分单个音乐片段，与人类判断相关性差
- **解决方案**: MUQ-EVAL
  - 基于 MuQ-310M 特征训练轻量级预测头
  - 使用 MusicEval 数据集（31个文本转音乐系统，专家质量评分）
- **性能**:
  - 系统级 SRCC = 0.957
  - 样本级 SRCC = 0.838
- **关键发现**: 
  - 冻结特征 + 注意力池化 + 两层 MLP 即达最佳
  - 编码器选择是主导设计因素
  - LoRA 适配模型仅需150个片段即可达到可用相关性
- **开源**: 代码、模型权重、评估脚本完全开源

**价值**: 首个开源、高性能、实时的 AI 音乐质量评估指标。

### 21. 多代理 LLM 架构在金融文档处理中的基准测试

**来源**: [arXiv:2603.22651](https://arxiv.org/abs/2603.22651) | **时间**: 2026-03-25

- **评估架构**: 4种多代理编排架构
  - 顺序流水线
  - 并行扇出合并
  - 层级主管-工作者
  - 反思自纠正循环
- **数据集**: 10,000份 SEC 文件（10-K、10-Q、8-K）
- **评估维度**: 25种提取字段类型（治理结构、高管薪酬、财务指标）
- **关键发现**:
  - 反思架构：最高 F1 = 0.943，但成本2.3倍
  - 层级架构：成本-准确性帕累托前沿最佳位置（F1 0.921，成本1.4倍）
  - 混合配置：以1.15倍成本恢复反思架构89%的准确性增益
- **扩展分析**: 1K到100K文档/天的扩展揭示非显而易见的吞吐量-准确性退化曲线

**实践价值**: 为在受监管金融环境中部署多代理 LLM 系统提供可操作指导。
