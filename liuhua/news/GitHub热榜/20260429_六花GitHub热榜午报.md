## 🔥 今日热榜 Top 项目

### 1. mattpocock/skills — AI Agent 实战技能库

- **链接**: https://github.com/mattpocock/skills
- **语言**: TypeScript
- **核心用途**: Matt Pocock（TypeScript 社区知名开发者）开放了自己日常使用的 Claude Code Agent 技能集合。涵盖规划与设计（PRD 生成、需求拆解、架构面试）、开发（TDD、Bug 追踪、代码架构改进）、工具配置（pre-commit hooks、Git 安全护栏）、知识管理（Obsidian 笔记、文章编辑）四大类共 15+ 个模块化技能。
- **为何值得关注**: 代表了 AI Agent 从"聊天助手"向"工程化工具链"演进的趋势。这些技能不是演示级别的 prompt，而是在生产环境日复一日使用的工程化工作流。`npx skills@latest add` 一行命令即可集成到 Claude Code 中。对 Agent 开发者来说是极佳的参考范本。
- **标签**: `#Agent技能` `#ClaudeCode` `#工程化` `#AI工具链`

### 2. abhigyanpatwari/GitNexus — 零服务器代码知识图谱引擎

- **链接**: https://github.com/abhigyanpatwari/GitNexus
- **语言**: TypeScript
- **核心用途**: 将任意代码仓库索引为知识图谱（依赖关系、调用链、执行流、聚类），通过 MCP 协议为 AI Agent 提供深度代码理解能力。支持 CLI + MCP 本地模式和 Web UI 浏览器模式，零服务器、零依赖。支持 Claude Code、Cursor、Codex、Windsurf、OpenCode 等主流编辑器。Tree-sitter 解析 + LadybugDB 存储，全本地运行，隐私安全。
- **为何值得关注**: 解决了 AI Agent 对大型代码库"只见树木不见森林"的核心痛点。传统 RAG 只能检索代码片段，GitNexus 构建完整的依赖/调用图谱，让 Agent 真正理解代码架构。自称"比 DeepWiki 更深"——DeepWiki 帮你理解代码，GitNexus 帮你分析代码。16 个 MCP 工具、多仓库支持、PR 爆炸半径分析等企业级功能。
- **标签**: `#AI编程` `#MCP` `#知识图谱` `#代码分析`

- **语言**: Python
- **核心用途**: Composio 团队维护的 OpenAI Codex 技能合集，涵盖开发工具（代码审查、CI 修复、MCP 构建）、生产力协作（Linear/Jira 管理、会议纪要、Notion 集成）、通信写作（邮件、简历生成）、数据分析（Datadog 日志、竞品分析）等 40+ 个实用技能。支持一键安装到 Codex CLI。

### 4. Michael-A-Kuykendall/shimmy — 轻量级本地 LLM 推理服务器

- **链接**: https://github.com/Michael-A-Kuykendall/shimmy
- **语言**: Rust
- **核心用途**: 纯 Rust 实现的 OpenAI API 兼容推理服务器，无需 Python 依赖。支持 GGUF + SafeTensors 格式，自动发现模型（HuggingFace 缓存、Ollama、本地目录），热模型切换，单二进制文件。v1.9.0 新增全 GPU 后端预编译二进制（CUDA/Vulkan/OpenCL/MLX）。支持 MoE 混合专家模型的 CPU/GPU 混合推理，可在消费级硬件运行 70B+ 模型。
- **为何值得关注**: Ollama 的轻量替代品，"免费且永远免费"。纯 Rust 无 Python 依赖，单二进制部署，对本地推理场景极具吸引力。MoE offloading 支持让消费级硬件也能跑大模型。MIT 开源协议。
- **标签**: `#本地推理` `#Rust` `#LLM服务器` `#OpenAI兼容`

### 5. gastownhall/beads — AI Agent 持久化记忆系统

- **链接**: https://github.com/gastownhall/beads
- **语言**: Go
- **核心用途**: 为编码 Agent 提供"记忆升级"。基于 Dolt（版本化 SQL 数据库）构建的分布式图谱化 Issue 追踪器，替代凌乱的 markdown 计划文件。支持依赖关系图、哈希 ID 零冲突、多层任务层级（Epic → Task → Sub-task）、语义记忆衰减（自动压缩旧任务）、隐身模式（不污染主仓库）。
- **为何值得关注**: 直击 AI Agent 长任务上下文丢失的痛点。不是简单的 TODO list，而是有版本控制、分支合并、多 Agent 协作能力的结构化记忆系统。Steve Yegge（知名技术博主/Google 前 20 号员工）参与的项目。支持 brew/npm/pip 多种安装方式。
- **标签**: `#Agent记忆` `#任务管理` `#Dolt` `#多Agent协作`

## 📡 GitHub 官方动态（4月27日）

### Copilot Cloud Agent 启动提速 20%
GitHub Copilot 云端 Agent 启动速度提升 20%+，得益于 GitHub Actions 自定义镜像优化预构建环境。继 3 月份 50% 提速后的持续优化，进一步缩短反馈循环。

### Copilot Code Review 6月1日起消耗 Actions 分钟数
GitHub 宣布从 2026 年 6 月 1 日起，Copilot Code Review 将同时消耗 AI Credits（按用量计费）和 GitHub Actions 分钟数（私有仓库）。公仓 Actions 分钟仍免费。影响 Pro/Pro+/Business/Enterprise 所有付费计划。

Student 计划中 GPT-5.3-Codex 从模型选择器中移除，但仍可通过自动模型选择使用。这是 Copilot 向基于用量计费过渡的一部分。

## 📊 趋势洞察

1. **Agent 技能生态爆发**: mattpocock/skills、awesome-codex-skills、beads 三个项目同时上榜，标志着 AI Agent 从"通用对话"向"专业化技能"转变。Agent 不再只是写代码，而是参与完整工程工作流。

3. **本地推理持续升温**: shimmy（Rust LLM 服务器）和 SillyTavern 持续上榜，反映开发者对本地/私有化 AI 推理的强烈需求。

4. **Agent 记忆与协作**: beads 项目代表了一个新方向——如何让 AI Agent 在长任务中保持上下文，多 Agent 间协调工作。
