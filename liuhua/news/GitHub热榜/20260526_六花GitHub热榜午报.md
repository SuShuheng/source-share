## 🔥 今日爆火项目

### 1. Understand-Anything — 代码库交互式知识图谱

- **仓库**: [Lum1104/Understand-Anything](https://github.com/Lum1104/Understand-Anything)
- **核心用途**: Claude Code 插件，将任意代码库转化为可交互的知识图谱。多 Agent 流水线扫描项目，提取文件、函数、类和依赖关系，生成可视化 Dashboard。
- **为何值得关注**: 支持 Claude Code / Codex / Cursor / Copilot / Gemini CLI 等 14 个平台一键安装。提供结构图、业务逻辑视图、引导式架构巡览、模糊搜索和语义搜索、Diff 影响分析等功能。MIT 开源，支持中文界面。今日 Trending 榜首，热度极高。
- **标签**: `#AI编程工具` `#代码理解` `#知识图谱`

### 2. AI Engineering from Scratch — 从零到一的 AI 工程课程

- **仓库**: [rohitg00/ai-engineering-from-scratch](https://github.com/rohitg00/ai-engineering-from-scratch)
- **核心用途**: 完整的 AI 工程师培养课程，435 节课、20 个阶段、约 320 小时。从线性代数到自主 Agent 群体，覆盖 Python / TypeScript / Rust / Julia 四种语言。每节课都以手写代码为起点，先推数学再写代码。
- **为何值得关注**: "84% 的学生已在用 AI 工具，但只有 18% 觉得准备好了"。这个课程填补了从"会用 API"到"理解底层原理"的鸿沟。MIT 开源，免费。非常适合系统性学习 AI 工程全栈。
- **标签**: `#AI教育` `#开源课程` `#从零开始`

### 3. Claude Code 官方插件目录

- **仓库**: [anthropics/claude-plugins-official](https://github.com/anthropics/claude-plugins-official)
- **核心用途**: Anthropic 官方维护的 Claude Code 高质量插件目录，包含内部插件和社区第三方插件。
- **为何值得关注**: Anthropic 官方下场做插件生态的标志性事件。一键安装 `/plugin install {plugin-name}@claude-plugins-official`。定义了标准插件结构（MCP 服务器、斜杠命令、Agent 定义、Skills），为 Claude Code 生态奠定基础。
- **标签**: `#ClaudeCode` `#插件生态` `#Anthropic官方`

### 4. CodeGraph — 编码 Agent 预索引知识图谱

- **仓库**: [colbymchenry/codegraph](https://github.com/colbymchenry/codegraph)
- **核心用途**: 为 Claude Code / Codex / Cursor 等 AI 编码 Agent 提供预索引的代码知识图谱——符号关系、调用图和代码结构。Agent 直接查图而非逐文件扫描。
- **为何值得关注**: 实测数据亮眼：平均 35% 成本降低、57% token 减少、46% 速度提升、71% 工具调用减少。在大型仓库（如 VS Code ~10k 文件）上效果最为显著——85% 工具调用减少。100% 本地运行，零数据外泄。
- **标签**: `#AI编程工具` `#性能优化` `#代码理解`

### 5. Multica — 开源托管 Agent 平台

- **仓库**: [multica-ai/multica](https://github.com/multica-ai/multica)
- **核心用途**: 将编码 Agent 变成真正的队友——像给同事分配 Issue 一样给 Agent 分配任务，Agent 自主执行、报告阻塞、更新状态。支持 Squad（Agent 小组）和技能复用。
- **为何值得关注**: 名字致敬 Multics 操作系统。核心理念是"Agent 是一等公民的队友"，而非工具。支持 Claude Code / Codex / Copilot CLI / OpenClaw 等 10+ 平台。"两个工程师 + Agent 舰队 = 二十个人的产出"。
- **标签**: `#Agent平台` `#团队协作` `#开源`

## 🌟 值得关注的项目

### 6. Pi — AI Agent 工具包

- **仓库**: [earendil-works/pi](https://github.com/earendil-works/pi)
- **核心用途**: AI Agent 工具包，包含交互式编码 Agent CLI、统一多供应商 LLM API（OpenAI/Anthropic/Google）、TUI 和 Web UI 库。
- **为何值得关注**: 统一的 LLM API 层是核心亮点，一套接口调用多家模型。自扩展的编码 Agent，支持发布会话数据到 Hugging Face 以改进编码 Agent 研究。MIT 开源。
- **标签**: `#Agent工具包` `#LLM统一API` `#编码Agent`

### 7. Karpathy Skills — 提升 Claude Code 行为的单文件

- **仓库**: [multica-ai/andrej-karpathy-skills](https://github.com/multica-ai/andrej-karpathy-skills)
- **核心用途**: 基于 Andrej Karpathy 对 LLM 编程陷阱的观察，提炼为四条原则的单文件 CLAUDE.md：先思考再编码、简单优先、精准修改、目标驱动执行。
- **为何值得关注**: Karpathy 指出 LLM 的核心问题——"模型会代替你做错误假设然后一路跑下去，不管理困惑、不寻求澄清"。这个文件是对症下药。极简、实用、即插即用。
- **标签**: `#Prompt工程` `#ClaudeCode` `#最佳实践`

### 8. Kronos — 金融市场基础模型

- **仓库**: [shiyu-coder/Kronos](https://github.com/shiyu-coder/Kronos)
- **核心用途**: 首个专为金融 K 线序列设计的开源基础模型。在 45+ 全球交易所数据上预训练，通过专用 tokenizer 将 OHLCV 数据量化为分层离散 token。
- **为何值得关注**: 已被 AAAI 2026 接收。不同于通用时序模型，专门针对金融数据高噪声特征设计。提供从 mini（4.1M）到 large（499.2M）的模型家族，均可在 Hugging Face 获取。支持实盘预测 Demo。
- **标签**: `#金融AI` `#基础模型` `#AAAI2026`

### 9. MiroFish — 群体智能预测引擎

- **仓库**: [666ghj/MiroFish](https://github.com/666ghj/MiroFish)
- **核心用途**: 下一代 AI 预测引擎，基于多 Agent 技术。从种子信息（新闻、政策、金融信号）自动构建高保真平行数字世界，成千上万的智能 Agent 在其中自由交互、社会演化，推演未来轨迹。
- **为何值得关注**: 将群体智能应用于预测的新范式。宏观层面可作为决策者的"排演实验室"，微观层面可作为创意沙盒。支持自然语言输入、自动生成预测报告。中国团队开发。
- **标签**: `#群体智能` `#预测引擎` `#多Agent仿真`

## 🛠️ 开发者工具精选

### 10. cmux — AI 编码 Agent 专用终端

- **仓库**: [manaflow-ai/cmux](https://github.com/manaflow-ai/cmux)
- **核心用途**: 基于 Ghostty 的 macOS 终端，专为 AI 编码 Agent 设计。垂直标签页、Agent 通知面板、内置浏览器、SSH 远程、Claude Code Teams 一键启动。
- **为何值得关注**: 解决了多 Agent 并行开发的核心痛点——当一个 Agent 需要你注意时，面板会亮起蓝色光环。内置浏览器让你的 Agent 可以在终端旁边直接打开网页。`cmux claude-teams` 一行命令启动 Claude Code 的 teammate 模式。
- **标签**: `#开发工具` `#终端` `#AI编码`

## 📰 GitHub 平台动态

### npm 分阶段发布正式上线 (5月22日)

- **来源**: [GitHub Changelog](https://github.blog/changelog/2026-05-22-staged-publishing-and-new-install-time-controls-for-npm)
- **要点**: npm 分阶段发布（Staged Publishing）正式 GA。包发布后不再立即可用，而需维护者显式批准。新增 `--allow-file`、`--allow-remote`、`--allow-directory` 安装源控制标志，强化供应链安全。需 npm CLI 11.15.0+。
- **意义**: npm 生态供应链安全的重要里程碑，建议所有 npm 包维护者关注。

### GitHub Issues 导航性能优化 (5月14日)

- **来源**: [GitHub Engineering Blog](https://github.blog/engineering/architecture-optimization/from-latency-to-instant-modernizing-github-issues-navigation-performance/)
- **要点**: GitHub Issues 团队使用客户端缓存（IndexedDB）、智能预取和 Service Worker，将问题页面导航延迟从秒级降至毫秒级。HPC（最高优先级内容）< 200ms 即为"即时"体验。
- **意义**: 对前端性能优化有参考价值的实战案例，可复用到任何数据密集型 Web 应用。
