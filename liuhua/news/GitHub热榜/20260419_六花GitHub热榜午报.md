## 热榜项目精选

### 1. EvoMap/evolver — AI Agent 自进化引擎

- **链接**：<https://github.com/EvoMap/evolver>
- **核心用途**：基于 GEP（Genome Evolution Protocol）的 AI Agent 自进化引擎。扫描运行日志，自动选择最优进化策略，生成协议约束的进化 prompt，并记录可审计的 EvolutionEvent。
- **为何值得关注**：将 Agent 的 prompt 调优从人工操作转变为可审计、可复用的自动化流程。支持策略预设（balanced / innovate / harden / repair-only），可离线运行，也可接入 EvoMap 网络进行技能共享和协作进化。本周从开源转向 source-available 模式（已发布的 MIT/GPL 版本仍可自由使用）。
- **分类**：AI Agent · 自进化

### 2. lsdefine/GenericAgent — 从 3K 行种子代码生长的自主 Agent

- **链接**：<https://github.com/lsdefine/GenericAgent>
- **核心用途**：极简自进化 Agent 框架。核心仅 ~3K 行代码，通过 9 个原子工具 + ~100 行 Agent Loop 实现对本地计算机的完整系统级控制（浏览器、终端、文件系统、键鼠输入、ADB 设备）。每次完成任务后自动将执行路径"结晶"为可复用的 skill，逐步积累个人技能树。
- **为何值得关注**：证明了"不要预加载技能——让 Agent 自行进化"的理念。支持 Claude / Gemini / Kimi / MiniMax 等多种模型，Token 消耗仅为同类 Agent 的 1/6（<30K context window）。整个仓库从 git init 到每个 commit message 都是 Agent 自主完成的。
- **分类**：AI Agent · 自进化

### 3. openai/openai-agents-python — OpenAI 多 Agent 工作流框架

- **链接**：<https://github.com/openai/openai-agents-python>
- **核心用途**：OpenAI 官方发布的轻量级多 Agent 工作流框架，支持 Agent 编排、任务分发与协作。
- **为何值得关注**：OpenAI 官方 Agent 框架，代表了大厂对多 Agent 协作范式的设计思路，是构建 AI Agent 应用的基础设施级项目。
- **分类**：AI Agent · 框架

### 4. z-lab/dflash — 块扩散投机解码加速

- **链接**：<https://github.com/z-lab/dflash>
- **核心用途**：基于块扩散（Block Diffusion）模型的投机解码（Speculative Decoding）方案，实现高效并行 draft 生成，加速大模型推理。
- **为何关注度**：已适配 Qwen3.5 系列、Kimi-K2.5、Llama-3.1 等主流模型，对推理速度提升显著。随着模型规模增长（122B、397B 参数），投机解码成为关键加速手段。附带完整训练 recipe（即将开源）。
- **分类**：AI · 推理加速

### 5. google/magika — AI 驱动的文件类型检测

- **链接**：<https://github.com/google/magika>
- **核心用途**：基于深度学习的文件内容类型检测工具。模型仅几 MB，单 CPU 推理约 5ms/文件，在 200+ 文件类型上达到 ~99% 准确率。
- **为何值得关注**：Google 内部大规模使用（Gmail、Drive、Safe Browsing 每周处理数千亿文件），已集成 VirusTotal 和 abuse.ch。支持 CLI（Rust）、Python、JS/TS、Go 多种语言绑定。论文发表于 IEEE/ACM ICSE 2025。
- **分类**：AI · 安全/工具

### 6. ChromeDevTools/chrome-devtools-mcp — Chrome DevTools MCP 服务器

- **链接**：<https://github.com/ChromeDevTools/chrome-devtools-mcp>
- **核心用途**：将 Chrome DevTools 能力暴露为 MCP 服务器，使 AI 编码助手（Gemini、Claude、Cursor、Copilot 等）能够控制和检查实时 Chrome 浏览器，实现自动化调试、性能分析和网络请求检查。
- **为何值得关注**：打通了 AI Agent 与浏览器 DevTools 的最后一公里。编码 Agent 不再只是写代码，还可以实时调试前端、分析性能瓶颈。支持 --slim 轻量模式和 headless 模式。
- **分类**：AI Agent · 开发工具

### 7. obra/superpowers — 编码 Agent 的完整开发方法论

- **链接**：<https://github.com/obra/superpowers>
- **核心用途**：为编码 Agent 提供完整的软件开发方法论——从需求澄清、设计评审、实施计划到子 Agent 驱动开发（subagent-driven-development），强调 TDD、YAGNI、DRY。
- **为何值得关注**：不只是一个 prompt，而是一套系统化的 Agent 工作流。已登陆 Claude Code 官方插件市场、OpenAI Codex、Cursor、GitHub Copilot 等主流平台。让编码 Agent 可以自主工作数小时不偏离计划。
- **分类**：AI Agent · 开发工具

### 8. pingdotgg/t3code — 编码 Agent 的极简 Web GUI

- **链接**：<https://github.com/pingdotgg/t3code>
- **核心用途**：为 Codex 和 Claude 编码 Agent 提供最小化 Web/桌面 GUI。支持 npx 一键启动，也有桌面应用（Windows/macOS/Linux）。
- **为何值得关注**：将命令行编码 Agent 包装为可视化界面，降低了使用门槛。t3 生态新作，由 ping（Theo 的 t3_stack 作者）出品。
- **分类**：开发工具

## GitHub 平台动态（来源：GitHub Changelog & Engineering Blog）

### gh skill 命令正式发布（4月16日）
GitHub CLI 新增 `gh skill` 命令，支持发现、安装、管理和发布 Agent Skills。遵循 [agentskills.io](https://agentskills.io) 开放规范，跨 Copilot、Claude Code、Cursor、Codex、Gemini CLI 等平台通用。引入版本锁定（`--pin`）、内容寻址变更检测和不可变发布等供应链安全机制。

### GitHub Copilot CLI 支持自动模型选择（4月17日）
Copilot CLI 现支持 `auto` 模式，动态路由到 GPT-5.4、GPT-5.3-Codex、Sonnet 4.6、Haiku 4.5 等模型，自动选择最优模型并缓解速率限制。付费用户使用 auto 可享 10% 折扣。

### GitHub 使用 eBPF 提升部署安全（4月16日）
GitHub 工程团队分享了如何利用 eBPF + cGroup 实现进程级网络过滤，防止部署脚本引入循环依赖。通过 `cilium/ebpf` Go 库实现，精准阻断仅部署脚本对 github.com 的网络访问，不影响生产流量。
