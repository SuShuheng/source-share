## 🔥 今日热榜项目

### 1. Claude HUD - Claude Code 状态增强插件
**项目地址**: [jarrodwatts/claude-hud](https://github.com/jarrodwatts/claude-hud)

**核心用途**: 为 Claude Code 提供 HUD（抬头显示）插件，实时显示上下文使用、活跃工具、运行中的 Agent 和待办事项进度。

**为何值得关注**:
- **提升开发体验**: 让你清楚知道 Claude Code 在做什么，避免上下文窗口过载
- **原生集成**: 使用 Claude Code 的 statusline API，无需额外窗口或 tmux
- **实时更新**: 每 300ms 刷新一次，显示模型名称、项目路径、git 分支、上下文使用率等
- **高度可配置**: 支持 Full/Essential/Minimal 三种预设，可自定义显示内容

**适用场景**: 重度使用 Claude Code 的开发者，特别是需要长时间会话、管理多个 Agent 任务的用户。

### 2. GET SHIT DONE (GSD) - 元提示与上下文工程系统
**项目地址**: [gsd-build/get-shit-done](https://github.com/gsd-build/get-shit-done)

**核心用途**: 轻量级但强大的元提示、上下文工程和规范驱动开发系统，支持 Claude Code、OpenCode、Gemini CLI、Codex、Copilot 和 Antigravity。

**为何值得关注**:
- **解决上下文腐烂**: 专门针对 Claude 上下文窗口填充后质量下降的问题
- **企业级工作流**: 支持 discuss → plan → execute → verify → ship 的完整流程
- **原子提交**: 每个任务独立提交，保持干净的 git 历史
- **跨平台支持**: 一键安装到多个 AI 编码助手
- **口碑极佳**: "如果你清楚知道自己要什么，这个工具真的能帮你构建出来。没有废话。"

**适用场景**: 希望用 AI 编码助手完成完整项目的开发者，特别是需要高质量代码和可维护性的团队。

### 3. Actions Runner Controller (ARC) 0.14.0
**项目地址**: [actions/runner-controller](https://github.com/actions/runner-controller)

**核心用途**: Kubernetes 原生的 GitHub Actions Runner 自动伸缩控制器。

**为何值得关注**:
- **多标签支持**: 单个 Runner Scale Set 可分配多个标签，简化组合配置
- **Scaleset 库客户端**: 使用 `actions/scaleset` 库，支持自定义自动伸缩方案
- **资源自定义**: 支持对内部资源（角色、服务账户、监听 Pod）添加自定义标签和注解
- **实验性 Helm Charts**: 更清晰的模板结构和 Docker Runner 配置

**适用场景**: 需要在 Kubernetes 上运行自托管 GitHub Actions Runner 的团队。

## 📢 GitHub 平台更新

### 1. Copilot Coding Agent 速度提升 50%
- **更新内容**: 优化了启动流程，Agent 开始工作速度提升 50%
- **影响**: 更快的反馈循环，PR 创建和迭代速度显著提升
- **适用对象**: Copilot Pro、Pro+、Business、Enterprise 订阅用户

### 2. GitHub Actions 时区支持
- **更新内容**: 定时工作流现支持 IANA 时区，不再受限于 UTC
- **用法**: 在 cron 表达式旁添加 `timezone: "America/New_York"` 等字段
- **意义**: 更直观的定时任务配置，减少时区转换错误

### 3. GitHub Actions 环境无需自动部署
- **更新内容**: 可使用环境的密钥和变量管理能力，而不创建部署记录
- **配置**: 在环境配置中添加 `deployment: false`
- **意义**: 简化 CI/CD 流程，减少不必要的部署历史

### 4. Copilot Coding Agent 会话可见性增强
- **内置设置步骤**: 显示仓库克隆和防火墙启动状态
- **自定义设置输出**: `copilot-setup-steps.yml` 的输出现在可见
- **子 Agent 活动**: 子 Agent 的活动默认折叠，可展开查看详情

### 5. Codespaces 数据驻留支持日本区域
- **更新内容**: 日本区域现支持 Codespaces 数据驻留（公测）
- **适用对象**: 需要数据驻留的 GitHub Enterprise 客户
- **可用区域**: 欧盟、澳大利亚、日本
