## 🔥 今日焦点

### 1. CodeGraph — 让 AI 编程助手拥有代码知识图谱

- **项目**: [colbymchenry/codegraph](https://github.com/colbymchenry/codegraph)
- **核心用途**: 为 Claude Code、Cursor、Codex CLI、OpenCode 等 AI 编程助手提供预索引的代码知识图谱。通过 MCP Server 交付符号关系、调用图和代码结构，替代昂贵的文件扫描探索。
- **亮点数据**: 7 个真实开源项目基准测试 — 平均 **35% 更便宜、59% 更少 token、49% 更快、70% 更少工具调用**。在 Tokio (Rust) 上节省 52% 成本；在 Excalidraw 上工具调用减少 86%。
- **为何值得关注**: AI 编程助手的 "上下文获取" 一直是 token 消耗大户。CodeGraph 通过本地 SQLite 索引 + FSEvents/inotify 实时同步，将探索型问答从 "扫文件" 变成 "查索引"，是大模型编程工具链的重要基础设施升级。支持 19+ 语言、13 个 Web 框架路由感知、100% 本地运行。
- **标签**: `#AI编程` `#MCP` `#代码智能`

### 2. Claude Plugins Official — Anthropic 官方插件市场

- **项目**: [anthropics/claude-plugins-official](https://github.com/anthropics/claude-plugins-official)
- **核心用途**: Anthropic 官方维护的 Claude Code 高质量插件目录。包含内部插件和第三方社区插件，支持 `/plugin install {name}@claude-plugins-official` 直接安装。
- **亮点**: 标准化插件结构（plugin.json + MCP 配置 + commands/agents/skills），第三方可提交插件审核上架。
- **为何值得关注**: 这标志着 Claude Code 正式建立了插件生态系统。插件市场 + 标准化结构 = AI 编程工具的平台化战略。对开发者和企业来说，这是构建 Claude Code 工作流扩展的官方入口。
- **标签**: `#Claude` `#插件生态` `#Anthropic`

### 3. oh-my-pi — IDE 级编程 Agent

- **项目**: [can1357/oh-my-pi](https://github.com/can1357/oh-my-pi)
- **核心用途**: 终端 AI 编程 Agent，内置 LSP（语言服务协议）、DAP（调试适配协议）、持久化 Python/Bun 运行时、浏览器工具、子代理系统。支持 40+ 模型提供商、32 个内置工具。
- **亮点**: 独创 hash-anchored 编辑格式，使 Grok Code Fast 1 的编辑成功率从 6.7% 提升至 68.3%；对 Gemini 3 Flash、MiniMax 等模型也有显著提升。~27k 行 Rust 核心。
- **为何值得关注**: 展示了 "工具链优化" 对 AI Agent 性能的巨大影响。同样的模型，不同的工具格式可以带来 10 倍性能差异。这是 Agent 工程而非模型规模的前沿实践。
- **标签**: `#AI编程` `#Agent` `#LSP`

### 4. Superpowers — AI 编程 Agent 的软件开发方法论

- **项目**: [obra/superpowers](https://github.com/obra/superpowers)
- **核心用途**: 为编程 Agent（Claude Code、Codex、Cursor、Gemini CLI 等）设计的完整软件开发方法论和可组合技能框架。自动触发需求澄清 → 规格确认 → 实现计划 → 子代理驱动开发 → 审查迭代流程。
- **亮点**: 强调真正的 TDD（红绿测试驱动）、YAGNI 原则、DRY 原则。已上线 Claude 官方插件市场。
- **为何值得关注**: 不只是工具，而是一套让 AI Agent 真正 "像资深工程师一样工作" 的方法论。自动化的子代理驱动开发可持续数小时不偏离计划，代表了 Agent 工作流编排的方向。
- **标签**: `#Agent方法论` `#TDD` `#工作流`

### 5. CLI-Anything — 让所有软件变成 Agent 原生

- **项目**: [HKUDS/CLI-Anything](https://github.com/HKUDS/CLI-Anything)
- **核心用途**: 自动为任何桌面/网页软件生成 CLI 包装器，让 AI Agent 能通过命令行操作这些软件。配有 CLI-Hub 社区市场（`pip install cli-anything-hub`）。
- **核心思想**: "今天的软件为人服务，明天的用户将是 Agent。" 将软件从 GUI-first 转变为 Agent-native。
- **为何值得关注**: Agent 要真正成为 "通用助手"，需要能操作所有软件。CLI-Anything 从工具层面解决了 Agent 与现有软件生态的互操作性问题。香港大学团队出品，学术+工程双重背书。
- **标签**: `#Agent原生` `#CLI` `#工具链`

### 6. AgentMemory — AI 编程 Agent 的持久记忆

- **项目**: [rohitg00/agentmemory](https://github.com/rohitg00/agentmemory)
- **核心用途**: 为 Claude Code、Cursor、Gemini CLI、Codex CLI、OpenClaw 等编程 Agent 提供持久化记忆服务。基于 iii 引擎，支持混合搜索、知识图谱、置信度评分和生命周期管理。
- **亮点**: 扩展了 Karpathy 的 LLM Wiki 模式，加入记忆检索基准测试。通过 MCP 协议接入，支持所有主流编程 Agent。
- **为何值得关注**: AI Agent 的 "健忘症" 是最大的痛点之一。AgentMemory 提供了跨会话、跨项目的记忆持久化方案，是 Agent 从 "单次对话工具" 走向 "长期开发伙伴" 的关键基础设施。
- **标签**: `#Agent记忆` `#MCP` `#持久化`

### 7. Andrej Karpathy Skills — 一份 CLAUDE.md 改善 LLM 编码行为

- **项目**: [multica-ai/andrej-karpathy-skills](https://github.com/multica-ai/andrej-karpathy-skills)
- **核心用途**: 基于 Andrej Karpathy 对 LLM 编码问题的观察，提炼为四条原则写入 CLAUDE.md：思考先于编码、简单优先、精准修改、目标驱动执行。
- **亮点**: 针对的问题极其精准 — LLM 会替你做错误假设、过度复杂化代码、删除它不理解的注释/代码。单文件方案，即插即用。
- **为何值得关注**: Karpathy 的观点直击 LLM 编码的核心痛点。这个项目把 "大佬经验" 变成了可复用的 Agent 指令，对每一个使用 AI 编程的人都实用。
- **标签**: `#LLM编码` `#Karpathy` `#Prompt工程`

### 8. AI Engineering from Scratch — 从零到一的 AI 工程课程

- **项目**: [rohitg00/ai-engineering-from-scratch](https://github.com/rohitg00/ai-engineering-from-scratch)
- **核心用途**: 435 节课、20 个阶段、约 320 小时的完整 AI 工程课程。从线性代数到自主 Agent 集群，覆盖 Python、TypeScript、Rust、Julia 四种语言。每节课产出一个可复用的 artifact。
- **亮点**: 不是碎片化的教程，而是从数学基础到生产级 Agent 的完整路径。强调 "先手写算法再引入框架"。
- **为何值得关注**: 这是目前 GitHub 上最系统化的 AI 工程学习资源之一。84% 的学生已在用 AI 工具但只有 18% 觉得准备好了 — 这个课程旨在弥补这一差距。MIT 开源协议。
- **标签**: `#AI教育` `#开源课程` `#全栈AI`
