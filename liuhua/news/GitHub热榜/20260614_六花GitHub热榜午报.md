## 📋 热榜项目清单

### 🥇 1. addyosmani/agent-skills
- **链接**: <https://github.com/addyosmani/agent-skills>
- **核心用途**: 为 AI 编程代理提供生产级工程技能（Skills）库
- **为何关注**: 作者 Addy Osmani（Google Chrome 团队知名工程师）。项目将软件工程最佳实践封装为 AI 代理可直接调用的「技能模块」，涵盖代码审查、测试策略、架构设计等。代表着 AI 辅助开发从「碎片化提示」走向「系统化工程能力」的趋势。
- **标签**: `AI Agent` `工程技能` `Skills Framework`

### 🥈 2. obra/superpowers
- **链接**: <https://github.com/obra/superpowers>
- **核心用途**: 代理技能框架 + 软件开发方法论
- **为何关注**: 不仅仅是技能集合，更是一套完整的「代理驱动开发方法论」。Claude AI 参与贡献代码，展示 AI 协作开发新模式。适合关注 Agent-as-Developer 范式的团队。
- **标签**: `AI Agent` `开发方法论` `Claude`

### 🥉 3. maziyarpanahi/openmed
- **链接**: <https://github.com/maziyarpanahi/openmed>
- **核心用途**: 本地优先的开源医疗 AI 平台，1000+ 医疗专用模型
- **为何关注**: **完全本地运行，数据不离开设备**。支持临床文本实体抽取、PII 去标识化，12 种语言，247 个 PII 检查点。基于 Apple MLX 框架，提供 Python SDK + 原生 Swift App。Apache-2.0 开源协议，已发 arXiv 论文。对医疗信息化和隐私保护领域具有里程碑意义。
- **标签**: `医疗AI` `本地推理` `隐私优先` `Apple MLX`

### 4. LMCache/LMCache
- **链接**: <https://github.com/LMCache/LMCache>
- **核心用途**: LLM 推理的 KV Cache 管理层
- **为何关注**: 将 KV Cache 从临时状态变为**可持久化、可跨引擎复用、可监控**的「AI 原生知识」。显著降低 TTFT（首 Token 延迟）并提升吞吐量，尤其对长上下文 Agent、多轮对话和 RAG 场景。厂商中立，兼容主流推理引擎。1713 commits，社区活跃度高。
- **标签**: `LLM推理优化` `KV Cache` `基础设施`

### 5. refactoringhq/tolaria
- **链接**: <https://github.com/refactoringhq/tolaria>
- **核心用途**: 桌面端 Markdown 知识库管理工具
- **为何关注**: 基于 Tauri 构建，3000+ commits，内置 MCP Server 支持。Claude AI 参与开发。定位为「Obsidian 的下一代竞争者」，将 AI 能力深度融入知识管理。适合关注 AI-Native 知识管理工具的开发者。
- **标签**: `知识管理` `Markdown` `Tauri` `MCP`

### 6. phuryn/pm-skills
- **链接**: <https://github.com/phuryn/pm-skills>
- **核心用途**: 产品经理 AI 技能市场 — 68 个 PM 技能 + 42 个链式工作流
- **为何关注**: 覆盖产品发现、策略、执行、上线、增长全生命周期。专为 Claude Code 和 Cowork 设计，兼容其他 AI 助手。是「AI 技能化」趋势在非工程领域（产品管理）的标杆项目。
- **标签**: `PM工具` `AI技能` `Claude Code`

### 7. msitarzewski/agency-agents
- **链接**: <https://github.com/msitarzewski/agency-agents>
- **核心用途**: 完整的 AI 代理团队 — 从前端到社区运营的各类专业代理
- **为何关注**: 每个代理都是具备个性、流程和交付物的专业专家。展示了「AI 团队编排」的实际落地方式，对构建多代理协作系统有参考价值。
- **标签**: `AI Agent` `多代理编排` `团队协作`

### 8. apple/container
- **链接**: <https://github.com/apple/container>
- **核心用途**: 在 Mac 上使用轻量级虚拟机创建和运行 Linux 容器
- **为何关注**: Apple 官方开源项目，纯 Swift 编写，针对 Apple Silicon 优化。为 macOS 开发者提供原生容器方案，无需 Docker Desktop。对 Mac 为主的开发团队意义重大。
- **标签**: `容器` `Apple Silicon` `Swift` `开发工具`

### 9. microsoft/PowerToys
- **链接**: <https://github.com/microsoft/PowerToys>
- **核心用途**: Windows 生产力工具集（窗口管理、颜色选取、键盘重映射等）
- **为何关注**: 微软官方开源的 Windows 效率神器，持续更新。虽非 AI 项目，但开发者使用率极高，长期占据 GitHub 热榜。
- **标签**: `Windows` `生产力工具` `微软官方`

### 10. music-assistant/server
- **链接**: <https://github.com/music-assistant/server>
- **核心用途**: 开源媒体库管理器，连接流媒体服务和智能音箱
- **为何关注**: 可在树莓派/NAS/Intel NUC 上常驻运行，整合多个流媒体服务。智能家居和音乐爱好者的高人气项目。
- **标签**: `智能家居` `媒体管理` `开源`
