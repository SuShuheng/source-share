## 🔥 今日焦点

### 1. mvanhorn/last30days-skill — AI Agent 多平台社交搜索引擎

- **链接**: https://github.com/mvanhorn/last30days-skill
- **标签**: `#1 Trending` `AI Agent` `社交搜索`
- **核心用途**: 一个 AI Agent Skill，跨 Reddit、X/Twitter、YouTube、HN、Polymarket 等十余个平台并行搜索，按真实用户互动（点赞、投票、真金白银）打分，AI 合成为一份简报。
- **为何值得关注**: v3 版本带来智能预搜索（自动解析人名→社交账号→社区）、HTML 简报导出、跨源聚类合并、"Best Takes" 幽默筛选等重磅功能。相当于给 AI Agent 装上了"社交信息雷达"。支持 Claude Code、Codex、Cursor、Copilot 等 50+ 宿主。

### 2. NousResearch/hermes-agent — 自我进化的 AI Agent

- **链接**: https://github.com/NousResearch/hermes-agent
- **标签**: `AI Agent` `自我学习` `多平台`
- **核心用途**: Nous Research 出品的自我进化 AI Agent。内置学习闭环——从经验中创建技能、使用中自我改进、跨会话构建用户模型。支持 Telegram/Discord/Slack/WhatsApp 等多平台。
- **为何值得关注**: 唯一内置"学习循环"的开源 Agent：自动技能创建、FTS5 会话搜索、Honcho 辩证用户建模。可在 $5 VPS 上运行，支持 200+ 模型切换。对于关注 Agent 自主能力演进的人来说，这是一个标杆项目。

---

### 3. RyanCodrai/turbovec — 8倍内存压缩的向量索引

- **链接**: https://github.com/RyanCodrai/turbovec
- **标签**: `Rust` `向量搜索` `RAG` `SIMD`
- **核心用途**: 基于 Google Research TurboQuant 算法的 Rust 向量索引，Python 绑定。1000万文档语料从 31GB（float32）压缩到 4GB，搜索速度超过 FAISS。
- **为何值得关注**: 无需训练、无需重建索引，在线 ingest。手写 NEON/AVX-512 SIMD 内核，支持搜索时 allowlist 过滤。已适配 LangChain、LlamaIndex、Haystack、Agno。对 RAG 场景的内存和延迟优化极具价值。

---

### 4. microsoft/pg_durable — PostgreSQL 原生持久化执行

- **链接**: https://github.com/microsoft/pg_durable
- **标签**: `PostgreSQL` `持久化执行` `工作流` `Microsoft`
- **核心用途**: 在 PostgreSQL 数据库内实现长时间运行、容错的 SQL 工作流函数。无需额外服务基础设施，定义 SQL 工作流，自动检查点，崩溃后自动恢复。
- **为何值得关注**: 微软出品，将 Durable Execution 模式直接嵌入 Postgres，消灭了 cron + worker + queue + 状态表的传统拼凑方案。已集成到 Azure HorizonDB。对数据工程师和 DBA 来说是一个范式转变。

## 🌟 值得关注

### 6. Leonxlnx/taste-skill — AI 生成 UI 的"反模板"框架

- **链接**: https://github.com/Leonxlnx/taste-skill
- **核心用途**: 便携 Agent Skill，升级 AI 生成的前端界面——更强的布局、排版、动效、间距，告别"AI 味"的千篇一律 UI。v2 版本引入三旋钮（VARIANCE / MOTION / DENSITY）调参系统。
- **为何值得关注**: 解决了 AI 前端生成的核心痛点——"看着就像 AI 做的"。支持 Claude Code、Codex、Cursor 等主流 Agent。

- **链接**: https://github.com/aaif-goose/goose
- **核心用途**: 通用 AI Agent，支持桌面 App + CLI + API，Rust 编写，15+ 模型提供商，70+ MCP 扩展。
- **为何值得关注**: 从 Block 迁入新成立的 Agentic AI Foundation (AAIF) @ Linux Foundation，标志着开源 AI Agent 治理模式的新探索。

### 8. lfnovo/open-notebook — 开源 NotebookLM 替代品

- **链接**: https://github.com/lfnovo/open-notebook
- **核心用途**: Google NotebookLM 的开源替代，支持 18+ AI 提供商、自托管、多模态内容、多播客生成、全文+向量搜索。
- **为何值得关注**: 隐私优先，数据完全自控。对比 NotebookLM 有更灵活的播客生成（1-4 位发言人）和完整 REST API。

- **链接**: https://github.com/refactoringhq/tolaria
- **核心用途**: macOS/Windows/Linux 桌面 App，管理 Markdown 知识库。Git-first、离线优先、AI 友好（支持 Claude Code/Codex/Gemini CLI）。
- **为何值得关注**: 由 Refactoring.fm 创始人 Luca Ronin 开发，用于管理其 10000+ 笔记的个人知识库。AI-First 但不 AI-Only，是 Agent 记忆管理的理想前端。

1. **Agent Skill 生态爆发**: last30days-skill、taste-skill、hermes-agent 均支持 `agentskills.io` 或 `npx skills` 安装标准，Agent 技能市场正在形成。
