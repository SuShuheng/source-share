## 🔥 今日爆火项目

### 1. NousResearch/hermes-agent — "The agent that grows with you"
- **链接**: https://github.com/NousResearch/hermes-agent
- **⭐ Stars**: ~146.8k | **语言**: Python | **协议**: MIT
- **核心用途**: 自我进化的 AI Agent 框架，内置学习闭环——从经验中创建技能、使用中自我改进、跨会话构建用户模型。支持 Telegram/Discord/Slack/WhatsApp/Signal 多平台。
- **为何值得关注**: Nous Research 出品，唯一一个在 GitHub 周榜和月榜同时出现的 AI Agent 框架。支持 200+ 模型切换，可在 $5 VPS 上运行，内置 cron 调度器和 Honcho 用户建模。Agent 领域的标杆级项目。

### 2. forrestchang/andrej-karpathy-skills — 单文件 CLAUDE.md
- **链接**: https://github.com/forrestchang/andrej-karpathy-skills
- **⭐ Stars**: ~126.4k | **New 2026**
- **核心用途**: 源自 Andrej Karpathy 对 LLM 编码陷阱的观察，整理为单个 CLAUDE.md 文件，用于改善 Claude Code 行为。
- **为何值得关注**: 以极简方式定义 AI 编码最佳实践，Star 增速惊人，已成为 AI 编码 Agent 生态的"规范参考"。一个文件解决 AI 编码的核心方法论问题。

### 3. mattpocock/skills — Skills for Real Engineers
- **链接**: https://github.com/mattpocock/skills
- **⭐ Stars**: ~74.5k | **New 2026**
- **核心用途**: 来自 Matt Pocock 的 .claude 目录，面向真实工程师的 AI 技能集合。
- **为何值得关注**: TypeScript 社区领袖直接分享的实战 AI 技能，一周内 +30.9k stars，说明开发者社区对高质量 Agent Skills 的渴求。

### 4. nexu-io/open-design — Claude Design 开源替代
- **链接**: https://github.com/nexu-io/open-design
- **⭐ Stars**: ~76.5k | **New 2026**
- **核心用途**: Anthropic Claude Design 的开源替代。Local-first、支持 16 种编码 Agent CLI 自动检测、31 个可组合技能、72 套品牌设计系统。可生成 Web/桌面/移动端原型、幻灯片、图片、视频。
- **为何值得关注**: Claude Design（2026-04-17 随 Opus 4.7 发布）引爆了 AI 设计话题，但保持闭源。Open Design 是其完整开源回应，支持 HTML/PDF/PPTX/MP4 导出，沙箱化预览。

### 5. google-labs-code/design.md — DESIGN.md 视觉身份规范
- **链接**: https://github.com/google-labs-code/design.md
- **⭐ Stars**: ~13.4k | **New 2026**
- **核心用途**: Google 推出的开放格式规范，用 YAML front matter + Markdown 描述视觉设计系统，让编码 Agent 持久理解项目的视觉身份。提供 lint、diff、export（Tailwind/DTCG）CLI 工具。
- **为何值得关注**: Google 官方出品，填补了"AI Agent 如何理解设计"的空白。`npx @google/design.md lint` 即可验证，可直接导出为 Tailwind 配置或 W3C DTCG 标准。AI + 设计工具链的关键基础设施。

## 🤖 AI 前沿项目

### 6. tinyhumansai/openhuman — 个人 AI 超级智能
- **链接**: https://github.com/tinyhumansai/openhuman
- **⭐ Stars**: ~2.1k | **New 2026** | **协议**: GNU
- **核心用途**: 开源桌面 AI Agent 助手。118+ 第三方集成（Gmail/Notion/GitHub/Slack 等）一键 OAuth，Memory Tree + Obsidian Wiki 本地知识库，桌面吉祥物可参加 Google Meet、语音交互。TokenJuice 压缩层节省 80% token 消耗。
- **为何值得关注**: 将 Agent 从"终端工具"进化为"桌面伙伴"，auto-fetch 每 20 分钟同步一次数据到本地 Memory Tree。Inspired by Karpathy's LLM Knowledgebase。

### 7. rohitg00/agentmemory — AI 编码 Agent 持久记忆
- **链接**: https://github.com/rohitg00/agentmemory
- **⭐ Stars**: ~5k | **New 2026**
- **核心用途**: 基于 iii 引擎的 AI 编码 Agent 持久记忆方案。兼容 Claude Code/Cursor/Gemini CLI/Codex CLI/Hermes/OpenClaw 等 15+ Agent，通过 MCP + hooks + REST API 共享记忆。
- **为何值得关注**: LongMemEval-S 基准测试 R@5 达 95.2%，超越 mem0（68.5%）和 Letta（83.2%）。年成本仅 $10，支持本地嵌入（$0）。一个命令 `npx @agentmemory/agentmemory` 即可启动。

### 8. cactus-compute/needle — 26M 参数微型函数调用模型
- **链接**: https://github.com/cactus-compute/needle
- **⭐ Stars**: ~93 | **New 2026**
- **核心用途**: 从 Gemini 3.1 蒸馏的 26M 参数"简单注意力网络"，专注于单次函数调用（function calling），可在手机、手表、眼镜等极小设备上运行。生产环境 prefill 6000 tok/s，decode 1200 tok/s。
- **为何值得关注**: 在单次函数调用任务上超越 FunctionGemma-270m、Qwen-0.6B、Granite-350m 等更大模型。AI 走向端侧的标志性项目——26M 参数即可驱动个人 AI 的工具调用能力。权重完全开放。

### 9. HKUDS/AI-Trader — 全自动 Agent 原生交易
- **链接**: https://github.com/HKUDS/AI-Trader
- **核心用途**: 100% 全自动的 Agent 原生交易系统。支持所有主流 AI Agent（OpenClaw/Claude Code/Codex/Cursor 等）。
- **为何值得关注**: 将 AI Agent 能力与金融交易深度结合，来自港大数据科学实验室。Agent 应用的垂直领域标杆。

## 🔧 高价值开发者工具

### 10. floci-io/floci — 免费 AWS 本地模拟器
- **链接**: https://github.com/floci-io/floci
- **⭐ Stars**: ~7.7k | **New 2026** | **协议**: MIT
- **核心用途**: 免费、开源的 AWS 本地模拟器，覆盖 47 个 AWS 服务。LocalStack 社区版 2026 年 3 月停更后的直接替代。Docker 原生构建，启动 ~24ms，空闲内存 ~13MiB。
- **为何值得关注**: LocalStack 社区版停更是 2026 年开发者工具生态的重大事件。Floci 以更轻量（镜像 ~90MB vs 1GB）、更快速、更完整的服务覆盖成为最佳替代。支持 API Gateway v2、Cognito、ElastiCache、RDS、Athena 等 LocalStack 免费版不支持的服务。

## 📡 GitHub 平台动态

### 🚀 GitHub Copilot App 技术预览版发布
- **来源**: GitHub Changelog | 2026-05-14
- **链接**: https://github.blog/changelog/2026-05-14-github-copilot-app-is-now-available-in-technical-preview
- **核心内容**: GitHub 原生桌面 Copilot 应用进入技术预览。支持从 Issue/PR/提示词启动会话，每个会话拥有独立分支和文件空间。可暂停/恢复会话、自动化重复工作流、集成终端和浏览器验证。支持 Agent Merge 自动处理 review 反馈。
- **意义**: 标志着 GitHub 从"代码补全工具"向"全栈 Agent 开发环境"的转型。

### ⚡ GitHub Issues 导航性能现代化
- **来源**: GitHub Engineering Blog | 2026-05-14
- **链接**: https://github.blog/engineering/architecture-optimization/from-latency-to-instant-modernizing-github-issues-navigation-performance/
- **核心内容**: GitHub Issues 团队使用 IndexedDB 客户端缓存 + 预热策略 + Service Worker，将导航感知延迟从"秒级"压缩到 <200ms（Instant 级别）。
- **意义**: AI 辅助工作流下，Issues 成为规划层，感知性能变得至关重要。这套模式可直接迁移到任何数据密集型 Web 应用。
