## 🔥 今日爆火项目 TOP 10

### 1. addyosmani/agent-skills — AI 编码 Agent 的生产级工程技能包

- **链接**：https://github.com/addyosmani/agent-skills
- **核心用途**：为 AI 编码 Agent 提供生产级工程技能（Skills），覆盖从需求定义（spec）到上线（ship）的全开发生命周期。包含 24 个结构化技能工作流和 7 个 slash 命令，支持 Claude Code、Cursor、Gemini CLI、Copilot 等主流 Agent。
- **为何值得关注**：由 Google Chrome DevRel 工程师 Addy Osmani 开源，将资深工程师的最佳实践编码为 AI Agent 可执行的工作流，每个技能都包含验证关卡和反合理化检查表。`/build auto` 模式可以一次批准后自主完成全流程，是 AI 编码协作方法论的重要参考。

### 2. obra/superpowers — Agent 技能框架与软件开发方法论

- **链接**：https://github.com/obra/superpowers
- **核心用途**：开源 Agent 技能框架，遵循 agentskills.io 标准，包含 14 个可组合的技能模块，提供完整的软件开发方法论。
- **为何值得关注**：与 addyosmani/agent-skills 形成互补，superpowers 更侧重框架层面的可组合性和方法论体系。两个项目同时登上热榜，标志着 **AI Agent 技能生态（Agent Skills）正在快速标准化**，这是 2026 年 AI 编码领域最值得关注的趋势之一。

### 3. NVIDIA/SkillSpector — AI Agent 技能安全扫描器

- **链接**：https://github.com/NVIDIA/SkillSpector
- **核心用途**：检测 AI Agent 技能（Skills）中的漏洞、恶意模式和安全风险。在安装 Agent 技能前进行安全审计，分析代码、依赖和隐藏元数据。
- **为何值得关注**：随着 Agent Skills 生态爆发（见前两项），**技能安全**成为刚需。NVIDIA 官方出品，说明安全维度已引起头部厂商高度重视。一个客户端的开发者想安装 AI "技能"加速工作，被 SkillSpector 扫出埋藏的恶意代码——这正是真实场景。

### 4. kenn-io/agentsview — AI 编码 Agent 本地分析仪表盘

- **链接**：https://github.com/kenn-io/agentsview
- **核心用途**：本地优先（local-first）的 Agent 会话搜索、分析、洞察和 Token 消耗统计工具。支持 Claude Code、Codex 等 20+ Agent，一个二进制文件，无需账号，数据完全本地化。
- **为何值得关注**：解决了 AI 编码 Agent 用户的核心痛点——**跨 Agent 的会话管理和成本可见性**。一键发现本机所有 Agent 会话，同步到本地 SQLite，提供 Web UI 和每日成本摘要。Docker 部署友好，支持 DuckDB 镜像和 PostgreSQL 后端。

### 5. LMCache/LMCache — 超高速 LLM KV Cache 层

- **链接**：https://github.com/LMCache/LMCache
- **核心用途**：高效 LLM KV Cache 管理层，跨请求复用注意力状态（KV Cache），避免重复计算长共享前缀，加速生成并降低 GPU/CPU 成本。兼容 vLLM 等推理引擎。
- **为何值得关注**：来自学术界（论文已上 arXiv）的工业级开源方案，是企业级 LLM 推理优化的关键基础设施。随着大模型推理成本成为核心瓶颈，KV Cache 复用技术直接影响生产环境的 TCO。

### 6. andrewyng/aisuite — 多模型统一调用接口

- **链接**：https://github.com/andrewyng/aisuite
- **核心用途**：轻量 Python 库，提供统一的 Chat Completions API 跨多家 Generative AI 供应商，并在其之上构建 Agents API（工具 + 工具包）。
- **为何值得关注**：Andrew Ng 出品，定位为 LLM 层的"统一接口"。两层架构（统一 Chat API + Agent API）让开发者可以零成本切换底层模型。对于需要避免供应商锁定的团队，这是首选方案之一。

### 7. x1xhlol/system-prompts-and-models-of-ai-tools — AI 工具系统提示词大全

- **链接**：https://github.com/x1xhlol/system-prompts-and-models-of-ai-tools
- **核心用途**：收集 28+ 主流 AI 编码工具（Cursor、Windsurf、Augment Code、Claude Code、Devin AI 等）的系统提示词全文，已超 134K Stars。
- **为何值得关注**：是研究 AI 工具设计思路的最佳参考库。通过对比不同工具的提示词架构，可以理解各家在安全边界、工具调用策略和上下文管理上的差异。**也对 AI 创业公司敲响安全警钟**——暴露的提示词可能成为攻击面。

### 8. apple/container — Apple 官方 Linux 容器工具

- **链接**：https://github.com/apple/container
- **核心用途**：在 Mac 上创建和运行 Linux 容器的工具，使用轻量级虚拟机，完全用 Swift 编写，针对 Apple Silicon 优化。
- **为何值得关注**：Apple 官方开源项目，基于 Containerization 框架为每个容器运行轻量 VM，**将安全隔离放在首位**。Docker 在 macOS 上的替代方案，原生 Apple Silicon 支持意味着性能开销极低。

### 9. chatwoot/chatwoot — 开源客户互动平台

- **链接**：https://github.com/chatwoot/chatwoot
- **核心用途**：开源、可自托管的客户互动套件，提供实时聊天、邮件支持、社交媒体整合等功能，是 Intercom/Zendesk 的替代方案。
- **为何值得关注**：持续保持高热度的成熟开源项目，最近新增了 AI 驱动的客服自动化能力。对于关注 **AI + 客服场景落地**的团队，代码库和社区都有很高参考价值。

### 10. swc-project/swc — Rust 编写的超快 JS/TS 编译器

- **链接**：https://github.com/swc-project/swc
- **核心用途**：Speedy Web Compiler，用 Rust 编写的超快 TypeScript/JavaScript 编译器，支持转译、压缩、类型检查等功能。
- **为何值得关注**：Next.js 等框架的底层核心引擎，持续保持活跃更新。在 AI 代码生成量暴增的时代，编译性能直接影响开发反馈循环速度。
