## 🔥 今日爆火项目

### 1. GitNexus — 零服务器代码智能引擎

- **仓库**: [abhigyanpatwari/GitNexus](https://github.com/abhigyanpatwari/GitNexus)
- **语言**: TypeScript
- **核心用途**: 将任意代码库索引为知识图谱，追踪每个依赖、调用链、功能集群和执行流，通过 MCP 协议为 AI Agent（Cursor、Claude Code、Codex、Windsurf 等）提供深层架构感知。
- **为何值得关注**:
  - 解决 AI 编程工具"不真正理解代码库结构"的痛点：传统 LLM 改了一个函数却不知道 47 个函数依赖它
  - 提供 16 个 MCP 工具（影响分析、跨文件重命名、流程追踪、Cypher 查询等）
  - 支持 14 种语言（TS/JS/Python/Java/Kotlin/C#/Go/Rust/PHP/Ruby/Swift/C/C++/Dart）
  - 全本地运行，零网络请求，隐私安全
  - Web UI 纯浏览器端运行（Tree-sitter WASM + LadybugDB WASM）
- **标签**: `#AI编程` `#知识图谱` `#MCP` `#代码智能`

### 2. Hermes Agent — 自我进化的 AI Agent

- **仓库**: [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent)
- **语言**: Python
- **核心用途**: Nous Research 打造的"能成长的"AI Agent。内置闭环学习：从经验中创建技能、使用中改进技能、自动持久化知识、搜索历史对话、跨会话构建用户模型。
- **为何值得关注**:
  - 唯一内置学习循环的 Agent：自主技能创建 + 技能自我改进 + Honcho 用户建模
  - 多平台覆盖：Telegram、Discord、Slack、WhatsApp、Signal、CLI
  - 支持 200+ 模型（OpenRouter、z.ai/GLM、Kimi/Moonshot、MiniMax、OpenAI 等）
  - 支持 $5 VPS 到 GPU 集群、Serverless 无服务器架构
  - 研究就绪：批量轨迹生成、Atropos RL 环境、轨迹压缩
  - MIT 开源协议
- **标签**: `#AI Agent` `#自我学习` `#多平台` `#Nous Research`

### 3. NVIDIA PersonaPlex — 全双工语音对话模型

- **仓库**: [NVIDIA/personaplex](https://github.com/NVIDIA/personaplex)
- **语言**: Python
- **核心用途**: 实时全双工语音对话模型，支持文本角色提示和音频语音条件的角色控制。基于 Moshi 架构和 Helium LLM，实现低延迟自然口语交互。
- **为何值得关注**:
  - NVIDIA 官方出品，7B 参数规模
  - 全双工实时语音：真正的"边听边说"能力
  - 角色控制：文本定义角色 + 音频定义声音，创造个性化 AI 角色
  - 丰富的预设语音（自然女声/男声 + 多样化女声/男声共 18 种）
  - 支持 GPU CPU 卸载，适合不同硬件配置
  - 附带完整 Web UI 和离线评估工具
- **标签**: `#语音AI` `#全双工` `#NVIDIA` `#实时对话`

### 4. Google AI Edge Gallery — 端侧 AI 体验中心

- **仓库**: [google-ai-edge/gallery](https://github.com/google-ai-edge/gallery)
- **语言**: Kotlin / Swift
- **核心用途**: Google 官方端侧 AI 体验应用，展示并允许用户在移动设备上本地运行开源 LLM。最新版本已支持 Gemma 4 系列。
- **为何值得关注**:
  - Google 官方出品，Android + iOS 双平台
  - 首个支持 Gemma 4 的端侧应用
  - 完全离线运行，数据不离开设备
  - 高性能生成式 AI 直接在手机硬件上运行
  - Apache 2.0 开源协议
- **标签**: `#端侧AI` `#Gemma4` `#Google` `#移动端`

### 5. Koharu — ML 驱动的漫画翻译器

- **仓库**: [mayocream/koharu](https://github.com/mayocream/koharu)
- **语言**: Rust
- **核心用途**: 本地优先的漫画翻译工具，结合目标检测、OCR、Inpainting 和 LLM 实现自动化漫画翻译。Tauri 桌面应用，全部组件用 Rust 编写。
- **为何值得关注**:
  - 全 Rust 技术栈：candle + llama.cpp 推理 + Tauri GUI
  - 完整流水线：文本区域检测 → OCR → Inpainting 去字 → LLM 翻译 → 垂直 CJK 文字渲染
  - GPU 加速：CUDA / Metal / Vulkan 全覆盖
  - PSD 分层导出，方便后期修图
  - 内置 MCP Server，支持 Agent 自动化翻译
  - 本地运行，不收集用户数据
- **标签**: `#漫画翻译` `#Rust` `#OCR` `#本地AI`

### 6. Ollama — 本地 LLM 运行平台（更新）

- **仓库**: [ollama/ollama](https://github.com/ollama/ollama)
- **语言**: Go
- **核心用途**: 一键在本地运行开源大模型。最新版本新增 Kimi-K2.5、GLM-5、MiniMax、DeepSeek、gpt-oss、Qwen、Gemma 等模型支持，以及 `ollama launch` 集成命令。
- **为何值得关注**:
  - 新增 `ollama launch` 命令：一键集成 Claude Code、Codex、Droid、OpenCode 等编程工具
  - 新增 `ollama launch openclaw` 命令：直接将 Ollama 变成跨平台 AI 助手
  - 持续扩大模型支持范围，中国模型厂商（Kimi、GLM、MiniMax、DeepSeek）全面接入
  - 仍是本地 LLM 运行的首选工具
- **标签**: `#本地LLM` `#Ollama` `#模型运行`
