## 🔥 今日焦点

### 1. GitHub Copilot 个人计划重大调整
**来源**: [GitHub Changelog](https://github.blog/changelog/2026-04-20-changes-to-github-copilot-plans-for-individuals) | 4月20日

GitHub 宣布暂停 Student / Pro / Pro+ 新用户注册，收紧各计划用量上限，Opus 模型从 Pro 移除（Pro+ 保留 Opus 4.7）。此举旨在保障现有付费用户体验稳定性。4月20日–5月20日可申请退款。

**为何关注**: 这直接影响了大量个人开发者使用 AI 编程助手的成本和可用模型，社区讨论热烈。

## 🤖 AI 热门项目

### 2. eyaltoledano/claude-task-master ⭐ 13.5k+
**链接**: https://github.com/eyaltoledano/claude-task-master
**核心用途**: AI 驱动的任务管理系统，可无缝嵌入 Cursor、Lovable、Windsurf、Roo 等 AI 编辑器。支持 MCP 协议，多模型提供商（Anthropic / OpenAI / Google / xAI / OpenRouter / Ollama），通过 `.env` 或 MCP 配置 API Key 即可运行。
**为何值得关注**: AI 编程工作流基础设施——把项目管理交给 AI Agent 自动拆解、追踪和执行，是当前最热门的 AI 编程方法论之一。

### 3. hexiecs/talk-normal ⭐ 762 (日增)
**链接**: https://github.com/hexiecs/talk-normal
**核心用途**: 一条 system prompt 让任何 LLM 「说人话」。消除 AI 回复中的套话、废话和过度格式化。经测试 GPT-4o-mini 字数减少 73%、GPT-5.4 减少 72%，同时保留全部有用信息。
**为何值得关注**: 直击 AI 交互的最大痛点之一——冗长回复。理念简洁，效果立竿见影，适用于所有模型。

### 4. nashsu/llm_wiki ⭐ 1.2k (日增)
**链接**: https://github.com/nashsu/llm_wiki
**核心用途**: 基于 Karpathy 的 LLM Wiki 模式打造的跨平台桌面应用。将文档自动转化为结构化、互链的知识库 Wiki。核心亮点：双步思维链摄取、4 信号知识图谱、Louvain 社区发现、Obsidian 兼容、Chrome Web Clipper。
**为何值得关注**: 解决了传统 RAG 每次从零检索的效率问题——知识编译一次、持续维护，是个人知识管理的重大升级。

### 5. yizhiyanhua-ai/fireworks-tech-graph ⭐ 2.2k (周增)

**为何值得关注**: 让 Claude Code 用户一键输出专业级技术图——AI 编程工作流中的文档可视化利器。

## 🛡️ 安全与基础设施

### 10. GitHub CodeQL 支持 Sanitizers & Validators
**来源**: [GitHub Changelog](https://github.blog/changelog/2026-04-21-codeql-now-supports-sanitizers-and-validators-in-models-as-data) | 4月21日

CodeQL 新增通过 YAML 数据扩展声明式定义 sanitizer 和 validator 的能力（`barrierModel` / `barrierGuardModel`），覆盖 C/C++、C#、Go、Java/Kotlin、JS/TS、Python、Ruby、Rust 全语言。CodeQL 2.25.2 起可用。

**为何关注**: 无需写自定义 CodeQL 代码即可扩展安全分析能力，大幅降低了代码安全的定制门槛。

### 11. GitHub 将弃用 HTTPS 中的 SHA-1
**来源**: [GitHub Changelog](https://github.blog/changelog/2026-04-20-sunsetting-sha-1-in-https-on-github) | 4月20日

时间表：2026年7月14日演习（Brownout），9月15日完全禁用。影响所有通过 HTTPS 访问 GitHub 的浏览器和 Git 客户端。使用现代浏览器即可自动兼容。
