## 🔥 今日爆火项目

### 1. anthropics/financial-services
- **链接**：<https://github.com/anthropics/financial-services>
- **语言**：Python | **星标**：14k+
- **核心用途**：Anthropic 官方发布的 Claude 金融服务业参考项目。包含完整的 Agent 体系（Pitch Agent、Market Researcher、GL Reconciler 等 9 个命名 Agent）、垂直领域插件（投行、股票研究、PE、财富管理）、以及 11 个 MCP 数据连接器（Daloopa、Morningstar、S&P Global、FactSet 等）。
- **为何值得关注**：这是目前 AI Agent 在金融领域落地的最完整参考实现。支持 Claude Cowork 插件和 Managed Agent API 两种部署方式，展示了如何将 LLM Agent 深度嵌入投资银行、股票研究、基金管理等真实业务流程。Anthropic 官方出品，含 Managed Agent 编排模板和子 Agent 委找示例，对 AI Agent 工程化实践极具参考价值。

### 2. LearningCircuit/local-deep-research
- **链接**：<https://github.com/LearningCircuit/local-deep-research>
- **核心用途**：本地化深度研究工具，SimpleQA 基准达 ~95%（如 Qwen3.6-27B 在 3090 上运行）。支持所有本地和云端 LLM（llama.cpp、Ollama、Google 等），集成 10+ 搜索引擎（arXiv、PubMed、私有文档），全程本地化加密。
- **为何值得关注**：在隐私和成本双重驱动下，本地化 RAG + 深度搜索工具需求爆发。该项目在有限硬件上实现了接近 SOTA 的表现，同时保留数据完全本地化的优势，对研究型用户和企业场景都极具吸引力。

### 3. InsForge/InsForge
- **链接**：<https://github.com/InsForge/InsForge>
- **核心用途**：基于 Postgres 的全栈后端平台，集成 Auth、Storage、Compute、Hosting、AI Gateway，专为 Coding Agent 设计。
- **为何值得关注**：定位为"AI 原生开发者的后端平台"，瞄准 AI Agent 需要的后端基础设施缺口。随着 AI Coding Agent 数量爆发式增长，一个为 Agent 优化的后端方案有明确的市场空间。

### 4. addyosmani/agent-skills
- **链接**：<https://github.com/addyosmani/agent-skills>
- **核心用途**：为 AI Coding Agent 提供生产级工程技能包。将工作流、质量门禁和最佳实践编码为 Agent 可直接使用的 Skill。
- **为何值得关注**：由知名工程师 Addy Osmani 发起。解决了当前 AI Coding Agent 的核心痛点——不只是写代码，而是按照工程标准写代码。Skill 体系化方案有望成为 Agent 工程能力的事实标准。

### 5. z-lab/dflash
- **链接**：<https://github.com/z-lab/dflash>
- **核心用途**：DFlash（Block Diffusion for Flash Speculative Decoding）——轻量级块扩散模型，用于推测解码加速，附带论文和模型权重。
- **为何值得关注**：将扩散模型思路引入 LLM 推测解码加速，属于前沿推理优化研究。如果效果验证，可能成为 LLM 推理加速的新范式。

## 🚀 AI 工具与框架

### 6. Hmbown/DeepSeek-TUI
- **链接**：<https://github.com/Hmbown/DeepSeek-TUI>
- **核心用途**：DeepSeek V4 的终端编程 Agent。支持流式推理输出、本地工作区编辑，从终端直接调用 `deepseek` 命令运行。
- **为何值得关注**：DeepSeek V4 的终端 Agent 实现，与 Claude Code、Cursor 等 IDE Agent 竞争。开源终端 Agent 方案，适合偏好 CLI 工作流的开发者。

### 7. VectifyAI/PageIndex
- **链接**：<https://github.com/VectifyAI/PageIndex>
- **核心用途**：无向量数据库、无分块的推理式 RAG 方案。通过文档索引和推理实现检索增强生成，抛弃传统 Embedding + 向量搜索路线。
- **为何值得关注**：挑战了"RAG 必须用向量数据库"的共识，用推理替代检索。如果效果可靠，将大幅简化 RAG 系统的工程复杂度。

### 8. vercel-labs/open-agents
- **链接**：<https://github.com/vercel-labs/open-agents>
- **核心用途**：Vercel 出品的云端后台编程 Agent 开源模板。包含 Web UI、Agent 运行时和部署方案。
- **为何值得关注**：Vercel 在前端和 Serverless 领域有深厚积累。该项目展示了如何在云平台上构建后台 Coding Agent，对 Agent 即服务（Agent-as-a-Service）的商业模式有示范意义。

### 9. decolua/9router
- **链接**：<https://github.com/decolua/9router>
- **核心用途**：免费无限 AI 编码路由器。连接 Claude Code、Codex、Cursor、Cline、Copilot 等工具到 40+ 免费模型提供商，支持自动回退、RTK 减少 40% Token。
- **为何值得关注**：降低 AI 编码工具的使用门槛，对个人开发者和学生极具吸引力。多 Provider 路由和自动回退设计也值得工程参考。

### 10. aaif-goose/goose
- **链接**：<https://github.com/aaif-goose/goose>
- **核心用途**：开源可扩展 AI Agent，从 block/goose 迁移至 Agentic AI Foundation。支持安装、执行、编辑、测试，可接入任意 LLM。
- **为何值得关注**：Goose 项目从 Block 公司独立为基金会运营，标志着开源 AI Agent 生态正在走向治理规范化。

## 📊 数据与生产力

### 11. PriorLabs/TabPFN
- **链接**：<https://github.com/PriorLabs/TabPFN>
- **核心用途**：表格数据基础模型（Foundation Model for Tabular Data）。用 Transformer 架构处理结构化/表格数据，无需特征工程。
- **为何值得关注**：AI 在结构化数据领域的突破性尝试。如果 TabPFN 在实际业务数据上表现稳定，将改变传统 ML 的建模方式。
