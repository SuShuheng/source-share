### 2. 🏆 colbymchenry/codegraph
**链接**: https://github.com/colbymchenry/codegraph
**核心用途**: 为 Claude Code、Codex、Cursor 等 AI 编程 Agent 提供**预索引代码知识图谱**，大幅减少 token 消耗和工具调用次数。
**为何值得关注**:
- 官方基准测试：平均 **35% 更便宜、59% token 减少、70% 工具调用减少**
- 支持 19+ 语言，100% 本地运行（SQLite），零外部依赖
- 解决了 AI Agent 在大型代码库中"迷路"的痛点——直接查询符号关系和调用图，而非逐文件 grep

### 3. 🔥 obra/superpowers
**链接**: https://github.com/obra/superpowers
**核心用途**: 完整的 AI 编程 Agent 开发方法论框架——从需求澄清、设计评审到子 Agent 驱动开发的全流程。
**为何值得关注**:

- 支持所有主流 Agent：Claude Code、Codex、Cursor、Copilot CLI、Gemini CLI 等
- Agent 可以自主工作数小时而不偏离计划——这对自动化生产力是质的飞跃

### 5. 🔥 multica-ai/andrej-karpathy-skills
**链接**: https://github.com/multica-ai/andrej-karpathy-skills
**核心用途**: 基于 Andrej Karpathy 对 LLM 编程弊端的观察，提炼成四条原则的单文件 CLAUDE.md。
**为何值得关注**:
- Karpathy 原话直指要害：模型"不管理困惑、不寻求澄清、不呈现权衡、该反驳时不反驳"
- 四条原则：**先思考再编码**、**简单优先**、**精确变更**、**目标驱动执行**
- 一个文件就能显著改善 Claude Code 的行为——低门槛高回报
