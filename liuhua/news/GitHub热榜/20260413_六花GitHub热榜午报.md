### 4. OpenBMB/VoxCPM
**🔗 [GitHub](https://github.com/OpenBMB/VoxCPM)** | 🌟 Trending #8

**核心用途：** 无Tokenizer的多语言文本转语音（TTS）系统，VoxCPM2支持30种语言、48kHz录音棚级音质。

**亮点：**
- 2B参数，基于MiniCPM-4骨干，200万+小时多语言语音数据训练
- Voice Design：仅凭自然语言描述（性别、年龄、语调、情感）创建全新声音
- 可控语音克隆：短参考音频+风格引导，保留原音色
- 实时流式推理：RTX 4090上RTF低至~0.3
- Apache 2.0开源，可商用
- 支持中国方言（粤语、吴语、闽南语等）

**为何值得关注：** TTS领域的重大突破。无需离散Tokenization的端到端扩散自回归架构，30种语言+中国方言支持，完全开源商用。刚刚发布即登顶Trending。

### 7. forrestchang/andrej-karpathy-skills
**🔗 [GitHub](https://github.com/forrestchang/andrej-karpathy-skills)** | 🌟 Trending #4

**核心用途：** 单个CLAUDE.md文件，将Andrej Karpathy对LLM编码陷阱的观察提炼为4条核心原则。

**四大原则：**
1. **先思考再编码** — 不要沉默假设，主动呈现多种解释
2. **简单优先** — 最小代码解决问题，不加功能、不加抽象
3. **精确手术式修改** — 只改必须改的，不动周边代码
4. **目标驱动执行** — 用成功标准替代命令式指令

**为何值得关注：** Karpathy亲自指出LLM编码的核心问题（乱假设、过度工程、随意删改）。这个项目用一句话概括：让AI编码少犯蠢。
