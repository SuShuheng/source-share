## 🔥 今日爆火项目 TOP 10

### 1. mvanhorn/last30days-skill — AI Agent 社交情报搜索引擎
- **链接**: https://github.com/mvanhorn/last30days-skill
- **核心用途**: AI Agent 技能插件，可跨 Reddit、X/Twitter、YouTube、HN、Polymarket、TikTok 等十余平台并行搜索，按真实用户互动（upvotes、likes、真金白银下注）评分排序，最终由 AI 合成简报。
- **为何值得关注**: GitHub Trending #1 当日仓库。v3 新增智能实体解析、跨来源聚类合并、HTML 简报导出、竞争对比模式。解决了"AI Agent 只能看到围墙花园内信息"的痛点，是 Agent 生态的重要基础设施。
- **标签**: `#AI-Agent` `#搜索` `#社交数据`

### 2. CopilotKit/CopilotKit — 全栈 Agent 前端框架 & AG-UI 协议
- **链接**: https://github.com/CopilotKit/CopilotKit
- **核心用途**: 构建 AI Agent 应用的前端 SDK，支持 React/Angular/Vue/React Native/Slack/Teams 等多平台。AG-UI 协议已被 Google、LangChain、AWS、Microsoft 等采用。
- **为何值得关注**: AI Agent 前端层的事实标准。支持 Generative UI（Agent 动态生成 UI 组件）、共享状态、Human-in-the-Loop、Self-Learning（CLHF）。从 React 库成长为多平台 Agent 框架。
- **标签**: `#AI-Agent` `#前端框架` `#AG-UI`

### 3. MemPalace/mempalace — 最佳基准测试开源 AI 记忆系统
- **链接**: https://github.com/MemPalace/mempalace
- **核心用途**: 本地优先的 AI 记忆系统，将对话历史以逐字文本存储，通过语义搜索检索。采用"宫殿"隐喻架构（人物→翼、主题→房间、内容→抽屉），支持范围化检索。LongMemEval R@5 达 96.6%。
- **为何值得关注**: 声称是"最佳基准测试的开源 AI 记忆系统"，零 API 调用，完全本地运行。可插拔后端（默认 ChromaDB），支持 MCP Server。对 AI Agent 长期记忆是刚需。
- **标签**: `#AI-记忆` `#本地优先` `#RAG`

### 4. danielmiessler/Personal_AI_Infrastructure — 个人 AI 生活操作系统
- **链接**: https://github.com/danielmiessler/Personal_AI_Infrastructure
- **核心用途**: PAI v5.0.0 — "Life Operating System"，统一的 Pulse 守护进程（生活仪表板），DA（数字助手）身份层，Algorithm v6.3.0，45 个技能、171 个工作流、37 个钩子。
- **为何值得关注**: 从 AI 脚手架升级为完整的生活操作系统。核心概念"理想状态（Ideal State）"——从当前状态到理想状态的系统化推进。适合个人知识管理和生活自动化。
- **标签**: `#AI-OS` `#个人效率` `#生活管理`

### 5. Panniantong/Agent-Reach — 给 AI Agent 装上互联网之眼
- **链接**: https://github.com/Panniantong/Agent-Reach
- **核心用途**: 一键为 AI Agent 安装互联网访问能力——Twitter、Reddit、YouTube、GitHub、B站、小红书、抖音、微博、微信公众号、V2EX 等全平台覆盖，零 API 费用。
- **为何值得关注**: 专为中文互联网生态设计，支持 B 站、小红书、抖音、微博、微信公众号等国内平台。兼容所有 Agent（Claude Code、OpenClaw、Cursor 等），一条命令安装，自带诊断工具。
- **标签**: `#AI-Agent` `#互联网接入` `#中文平台`

### 6. openai/plugins — OpenAI Codex 插件集合
- **链接**: https://github.com/openai/plugins
- **核心用途**: OpenAI Codex 插件示例合集，包含 Figma、Notion、iOS/macOS/Web 应用构建、Expo、Netlify 等官方插件，每个插件含 manifest + skills + MCP 配置。
- **为何值得关注**: OpenAI 官方仓库，展示了 Codex 插件生态的完整结构。对开发者构建自己的 Codex 插件有直接参考价值。
- **标签**: `#OpenAI` `#Codex` `#插件生态`

### 7. obra/superpowers — Agent 驱动软件开发方法论
- **链接**: https://github.com/obra/superpowers
- **核心用途**: 完整的 Agent 软件开发方法论框架——从需求澄清→规格确认→实现计划→子 Agent 并行开发→代码审查，强调 TDD、YAGNI、DRY。
- **为何值得关注**: 不只是代码工具，而是一套让 AI Agent 像工程团队一样工作的方法论。支持 Claude Code、Codex CLI、Gemini CLI、Cursor 等。子 Agent 驱动开发（Subagent-Driven Development）是亮点。
- **标签**: `#AI-开发` `#方法论` `#Agent-SDLC`

### 8. microsoft/VibeVoice — 开源前沿语音 AI
- **链接**: https://github.com/microsoft/VibeVoice
- **核心用途**: 微软开源语音 AI 家族，包含 ASR（语音识别，支持 50+ 语言，60 分钟长音频一次转录）和 Realtime TTS（流式文本转语音，0.5B 模型）。
- **为何值得关注**: ASR 已集成 Hugging Face Transformers。核心技术是 7.5Hz 超低帧率连续语音 tokenizer。ICLR 2026 Oral 论文。支持 vLLM 推理加速。
- **标签**: `#语音AI` `#ASR` `#TTS` `#微软开源`

### 9. microsoft/mxc — 策略驱动沙箱代码执行系统
- **链接**: https://github.com/microsoft/mxc
- **核心用途**: 跨平台沙箱执行系统，用于安全运行不可信代码（模型输出、插件、工具）。支持 ProcessContainer、Windows Sandbox、LXC、Hyperlight 等多种隔离后端。
- **为何值得关注**: Microsoft Execution Containers 的早期预览版。统一的 JSON 配置 + TypeScript SDK。对 AI Agent 代码执行安全至关重要——Agent 生成代码的安全沙箱是核心基础设施。
- **标签**: `#安全` `#沙箱` `#代码执行` `#微软`

### 10. lfnovo/open-notebook — 开源版 NotebookLM
- **链接**: https://github.com/lfnovo/open-notebook
- **核心用途**: Google NotebookLM 的开源替代品，支持 18+ AI 提供商（OpenAI、Anthropic、Ollama 等），多模态内容（PDF/视频/音频/网页），1-4 人播客生成，全文+向量搜索。
- **为何值得关注**: 数据自主可控，自托管。比 NotebookLM 更灵活的 AI 提供商选择和多语言 UI（含中文）。播客功能支持自定义说话人配置。
- **标签**: `#知识管理` `#NotebookLM替代` `#自托管`
